# 📔 오답 노트: BOJ 1629 곱셈

## 1. 문제 정보
- **번호**: 1629
- **이름**: 곱셈 (Multiplication)
- **난이도**: Silver I
- **날짜**: 2026-04-07
- **소요 시간**: 약 15분

## 2. 핵심 아이디어 (Key Concept)
- **분할 정복을 이용한 거듭제곱 (Exponentiation by Squaring)**: $A^{100}$을 구할 때 $A^{50} \times A^{50}$으로 나누어 계산하여 계산 횟수를 획기적으로 줄임 ($O(\log N)$).
- **모듈러 연산의 성질**: $(A \times B) \pmod C = ((A \pmod C) \times (B \pmod C)) \pmod C$ 를 이용하여 중간 계산 결과가 너무 커지지 않게 유지함.

## 3. 내가 놓쳤던 부분 / 주의할 점
- 그냥 `A**B % C`를 하면 숫자가 너무 커져서 파이썬에서도 처리 속도가 느려질 수 있고, 다른 언어에서는 오버플로우가 발생함.
- 반드시 재귀나 반복문을 통해 절반씩 나누어 계산해야 함.
- 거듭제곱 횟수(B)가 홀수일 때 마지막에 `A`를 한 번 더 곱해주는 처리가 필요함.

## 4. 최종 풀이 코드 (Python)
```python
import sys

def power(a, b, c):
    if b == 0:
        return 1
    if b == 1:
        return a % c
    
    half = power(a, b // 2, c)
    # 지수가 짝수일 때: (A^(B//2))^2
    if b % 2 == 0:
        return (half * half) % c
    # 지수가 홀수일 때: (A^(B//2))^2 * A
    else:
        return (half * half * a) % c

# 입력 예시: 10 11 12
# 결과: 4
```

## 5. 새로 배운 점
- 거대한 지수(B)를 다룰 때는 항상 '절반씩 나누기'를 생각하자.
- 파이썬의 `pow(A, B, C)` 함수도 내부적으로 이 알고리즘을 사용한다는 것을 알게 됨.
