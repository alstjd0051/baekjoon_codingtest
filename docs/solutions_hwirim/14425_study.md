# BOJ 14425 - 문자열 집합

## 문제 분석 및 풀이 전략

이 문제는 주어진 문자열 집합 내에 특정 문자열들이 포함되어 있는지를 확인하고 그 횟수를 세는 전형적인 탐색 문제인니다. 데이터의 규모와 탐색 효율성을 고려하여 최적의 자료구조를 선택하는 과정이 핵심입니다.

### 연역적 사고 과정

1. 문제의 본질 파악: 집합 $S$에 속한 $N$개의 문자열이 주어지고, 이후 주어지는 $M$개의 쿼리 문자열 각각에 대해 $S$에 포함되는지 여부를 판단해야 합니다.

2. 선형 탐색의 한계: 만약 집합 $S$를 단순 리스트로 관리한다면, 하나의 쿼리를 처리할 때마다 $O(N)$의 시간이 소요됩니다. 전체 과정은 $O(M \times N \times L)$ (여기서 $L$은 문자열의 길이)이 되며, $N, M$이 각각 10,000일 경우 최대 $10^8$ 수준의 연산이 필요하여 비효율적입니다.

3. 탐색 효율화 (자료구조 선택):
   - 해시 테이블(Hash Table): 파이썬의 `set` 자료구조는 내부적으로 해시 테이블을 사용합니다. 특정 원소의 존재 여부를 평균 $O(1)$의 시간에 확인할 수 있습니다. 이 경우 전체 시간 복잡도는 $O((N+M) \times L)$로 획기적으로 단축됩니다.
   - 트라이(Trie): 문자열 탐색 전용 트리 구조로, 문자열 길이에 비례하는 $O(L)$ 탐색 속도를 보장합니다. 하지만 이 문제에서는 단순 포함 여부만 확인하면 되므로, 구현이 더 단순하고 메모리 효율이 양호한 해시 기반의 `set`을 선택하는 것이 연역적으로 타당합니다.

4. 최종 결론: $N$개의 문자열을 파이썬의 `set`에 저장한 뒤, $M$개의 문자열을 순회하며 `if ... in set` 연산을 수행하여 카운트를 누적하는 방식이 가장 최적화된 풀이 경로입니다.

### 구현 세부 사항

- `sys.stdin.readline`을 사용하여 대량의 문자열 입력을 빠르게 처리합니다.
- 첫 번째 그룹($N$개)은 `set` 자료구조에 저장하여 중복을 자동 제거(문제에서 주어지진 않으나 안전장치)하고 빠른 조회를 가능하게 합니다.
- 두 번째 그룹($M$개)은 반복문을 돌며 존재 여부만 확인합니다.

## 복잡도 분석

- 시간 복잡도: $O((N + M) \times L)$
  - 집합 $S$를 생성하는 데 $O(N \times L)$, 각 쿼리를 확인하는 데 $O(M \times L)$이 소요됩니다 ($L$은 문자열의 최대 길이).
- 공간 복잡도: $O(N \times L)$
  - 집합 $S$의 모든 문자열을 메모리에 저장해야 합니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O((N+M) \times L)$)의 도출:
   문자열의 존재 여부를 파악하는 해시 테이블(`set`)의 조회 시간은 평균적으로 $O(1)$입니다. 그러나 문자열 비교 및 해시값 계산은 문자열의 길이 $L$에 비례하는 시간이 필요합니다. 따라서 $N$개의 문자열을 집합에 추가하는 데 $O(N \times L)$, $M$개의 문자열에 대해 존재 여부를 조회하는 데 $O(M \times L)$이 각각 소요됩니다. 이를 합산하면 전체 입력 데이터의 크기에 비례하는 $O((N+M) \times L)$의 시간 복잡도가 필연적으로 도출됩니다.

2. 공간 복잡도($O(N \times L)$)의 도출:
   우리는 집합 $S$에 포함된 모든 문자열 정보를 해시 테이블에 유지해야 합니다. 저장하는 문자열의 개수가 $N$개이고 각 문자열의 평균 길이가 $L$이라고 할 때, 이들을 저장하기 위한 공간은 문자열 본문의 총 합계인 $O(N \times L)$이 됩니다. 추가적으로 해시 테이블 관리를 위한 약간의 오버헤드가 발생하지만, 점근적 표기법상 수용 가능한 범위 내에 있으므로 주된 메모리 사용량은 집합 데이터의 크기에 비례한다고 판단할 수 있습니다.
