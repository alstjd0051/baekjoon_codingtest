# BOJ 5430 - AC

## 문제 분석 및 풀이 전략

이 문제는 정수 배열에 대해 수행할 수 있는 두 가지 연산 R(뒤집기)과 D(버리기)를 효율적으로 구현하는 문제입니다. 연산의 횟수와 배열의 크기가 상당히 크므로, 매번 실제로 배열을 뒤집는 연산을 수행하면 시간 초과가 발생할 수 있습니다.

### 연역적 사고 과정

1. 연산 성능 분석:
   - R(뒤집기): 배열의 순서를 실제로 바꾸는 작업은 $O(N)$의 시간이 소요됩니다. 연산 명령어의 길이 $P$가 최대 100,000이므로, 최악의 경우 $O(P \times N)$의 시간이 걸려 제한 시간 내에 해결이 불가능합니다.
   - D(버리기): 배열의 첫 번째 원소를 제거하는 작업은 일반적인 리스트에서 $O(N)$이 소요되지만, 적절한 자료구조(Deque)를 사용하면 $O(1)$로 단축할 수 있습니다.

2. 해결책 도출 (가상 뒤집기):
   - 배열을 실제로 뒤집는 대신, 현재 배열의 방향(정방향 또는 역방향)을 나타내는 상태 변수(`reverse_flag`)를 관리합니다.
   - R 연산이 들어오면 `reverse_flag`의 상태를 반전시킵니다.
   - D 연산이 들어올 때:
     - `reverse_flag`가 거짓(정방향)이면 앞에서 원소를 제거합니다.
     - `reverse_flag`가 참(역방향)이면 뒤에서 원소를 제거합니다.

3. 자료구조 선택:
   - 양방향에서 삽입과 삭제가 효율적인 덱(Deque) 자료구조를 사용합니다. 이를 통해 D 연산을 방향에 관계없이 $O(1)$에 처리할 수 있습니다.

4. 예외 및 최종 처리:
   - 빈 배열에서 D 연산을 수행하려 할 때 "error"를 출력하도록 설계합니다.
   - 모든 연산이 끝난 후, `reverse_flag`가 참인 상태라면 덱에 남은 원소들을 한 번만 뒤집어 최종 결과를 출력합니다.

### 구현 세부 사항

- 입력 형식 처리: `[x1,x2,...]` 형태의 문자열을 정수 리스트로 변환하기 위해 문자열 슬라이싱과 `split(',')`을 활용합니다.
- 빈 배열 입력 처리: `n=0`인 경우 `[]`가 입력되는데, 이를 적절히 처리하여 비어있는 Deque을 생성합니다.

## 복잡도 분석

- 시간 복잡도: $O(P + N)$
  - 연산 명령어 문자열 $P$를 한 번 순회하며, 각 연산은 덱을 통해 $O(1)$에 처리됩니다. 최종 출력 시 최대 한 번 배열을 뒤집으므로 $O(N)$이 추가됩니다.
- 공간 복잡도: $O(N)$
  - 입력받은 $N$개의 정수를 덱에 저장해야 합니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(P + N)$)의 도출:
   명령어 문자열의 길이를 $P$, 배열의 길이를 $N$이라 할 때, 명령어 하나를 해석하고 덱에서 원소를 제거하는 작업은 방향과 관계없이 $O(1)$입니다. 따라서 $P$번의 명령어를 처리하는 데 $O(P)$가 소요됩니다. 마지막에 남은 원소들을 문자열로 변환하여 출력하는 과정($O(N)$)과 뒤집는 과정($O(N)$)을 포함하면 전체 복잡도는 $O(P + N)$이 됩니다. 이는 최악의 경우 약 70만 번의 연산으로, 1초라는 제한 시간을 충실히 준수할 수 있습니다.

2. 공간 복잡도($O(N)$)의 도출:
   입력받은 수들을 덱에 보관하기 위한 메모리가 주된 사용량입니다. $N$개의 원소를 저장하므로 $O(N)$의 공간을 사용합니다. 명령어 문자열을 저장하는 $O(P)$의 공간도 필요하지만, 결과적으로 입력 데이터의 총량에 비례하는 선형 공간 복잡도를 가집니다.
