# BOJ 3568 - iSharp

## 문제 분석
- 독특한 변수 선언 방식(공통 타입 + 개별 추가 타입/변수명)을 표준적인 개별 선언 방식으로 변환하는 문제입니다.
- 추가적인 변수형(`[]`, `&`, `*`)은 변수명 오른쪽에 붙어 있으며, 이를 왼쪽으로 옮길 때는 순서를 뒤집어야 합니다.
- 단, 배열(`[]`)은 하나의 단위로 취급하여 뒤집힙니다.

## 접근 방법
- **문자열 파싱**:
    - 입력받은 줄에서 마지막 세미콜론을 제거하고 공백과 콤마를 기준으로 분리합니다.
    - 첫 번째 요소는 공통 타입(`base_type`)이 됩니다.
- **개별 변수 처리**:
    - 각 변수 문자열에서 알파벳(변수명)과 나머지(추가 타입)를 분리합니다.
    - 추가 타입을 뒤에서부터 읽으며 `base_type` 뒤에 붙여줍니다. 이때 `]`를 만나면 `[]`로 처리하여 인덱스를 2씩 줄입니다.
- **결과 출력**:
    - `공통타입 + 뒤집힌타입 + 변수명 + ;` 형식으로 출력합니다.

## 코드 설명
- 순회하면서 `isalpha()`를 기준으로 이름과 타입을 구분합니다.
- 타입을 역순으로 순회하며 `[]` 단위를 유지하여 `reversed_extras`를 생성합니다.
