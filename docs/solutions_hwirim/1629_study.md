# BOJ 1629 - 곱셈

## 문제 분석 및 풀이 전략

이 문제는 매우 큰 지수를 가진 거듭제곱 연산을 효율적으로 수행하고, 그 결과를 특정 수로 나눈 나머지를 구하는 문제입니다. 단순 반복문을 통한 계산은 시간 초과를 발생시키므로, 분할 정복(Divide and Conquer)을 이용한 지수 법칙의 활용이 필수적입니다.

### 연역적 사고 과정

1. 기본 연산의 한계: $A^B$를 구하기 위해 $A$를 $B$번 곱하는 연산은 $O(B)$의 시간 복잡도를 가집니다. $B$가 최대 $2,147,483,647$이므로 이 방식은 제한 시간 내에 해결이 불가능합니다.

2. 문제의 축소 (분할 정복): 지수 법칙 $A^{n+m} = A^n \times A^m$을 활용합니다. 특히 $B$를 절반으로 나누어 $A^B = A^{B/2} \times A^{B/2}$로 정의하면, 하위 문제($A^{B/2}$)를 한 번만 계산하여 그 결과를 재사용함으로써 총 계산 횟수를 $O(\log B)$로 단축할 수 있습니다.

3. 지수의 홀짝성 처리:
   - $B$가 짝수일 때: $A^B = (A^{B/2})^2$
   - $B$가 홀수일 때: $A^B = (A^{B/2})^2 \times A$
   이와 같이 모든 경우를 $B/2$에 대한 문제로 귀납적으로 정의할 수 있습니다.

4. 수의 범위 관리 (모듈러 연산): $A^B$는 상상을 초월하는 큰 수가 될 수 있어 메모리와 처리 속도에 악영향을 줍니다. 모듈러 연산의 성질 $(X \times Y) \pmod C = ((X \pmod C) \times (Y \pmod C)) \pmod C$를 이용하여, 모든 중간 곱셈 단계마다 나머지를 취함으로써 숫자의 크기를 $C$ 미만으로 일정하게 유지합니다.

5. 기저 사례(Base Case) 설정: 지수 $B$가 $1$인 경우 $A \pmod C$를 반환하고, $B$가 $0$인 경우 $1 \pmod C$를 반환하도록 설계하여 재귀 호출을 종료합니다.

### 구현 세부 사항

- 재귀 함수 설계: `power(a, b, c)` 함수를 만들어 내부에 분할 정복 로직을 구현합니다.
- 효율성 유지: `half = power(a, b // 2, c)`와 같이 결과값을 변수에 저장하여 동일한 재귀 호출이 중복 발생하지 않도록 합니다.

## 복잡도 분석

- 시간 복잡도: $O(\log B)$
  - 매 호출마다 지수가 절반으로 줄어들므로, 전체 호출 횟수는 지수의 비트 수에 비례합니다.
- 공간 복잡도: $O(\log B)$
  - 재귀 호출의 깊이가 지수의 로그 값에 비례하여 스택 프레임이 생성됩니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(\log B)$)의 도출:
   우리는 지수 $B$를 밑으로 하는 거듭제곱을 구해야 합니다. 단순 곱셈 대신 지수를 절반씩 나누는 전략을 택했으므로, 한 번의 재귀 단계마다 문제의 규모($B$)가 $1/2$로 감소합니다. 이는 수학적으로 $B \times (1/2)^x = 1$이 되는 $x$를 찾는 과정과 같으며, $x = \log_2 B$가 됩니다. 따라서 전체 연산 횟수는 지수 $B$의 크기에 로그를 취한 값에 비례하는 $O(\log B)$가 됨을 논리적으로 도출할 수 있습니다.

2. 공간 복잡도($O(\log B)$)의 도출:
   이 풀이는 재귀(Recursion) 방식을 사용합니다. 컴퓨터 시스템에서 재귀 호출은 호출될 때마다 독립적인 메모리 영역(스택 프레임)을 차지합니다. 위에서 도출한 대로 재귀 호출이 $B$가 $1$이 될 때까지 $\log_2 B$번 발생하므로, 메모리 상에 동시에 쌓이는 스택의 최대 깊이 역시 $\log_2 B$에 수렴하게 됩니다. 따라서 이 알고리즘의 보조적인 공간 사용량은 $O(\log B)$가 됩니다.
