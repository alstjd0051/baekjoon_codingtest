# BOJ 9012 - 괄호

## 문제 분석 및 풀이 전략

이 문제는 주어진 괄호 문자열이 올바른 구성(VPS)인지 판단하는 전형적인 스택(Stack) 활용 문제 또는 계수(Counting) 문제입니다.

### 연역적 사고 과정

1. 문제의 정의 규명: 올바른 괄호 문자열(VPS)이 되기 위해서는 모든 열린 괄호 '('가 적절한 위치에서 닫힌 괄호 ')'와 짝을 이루어야 합니다.

2. 순차적 처리의 정당성: 문자열을 왼쪽에서 오른쪽으로 한 글자씩 확인할 때, 어떤 시점이든 ')'의 개수가 지금까지 나온 '('의 개수를 초과해서는 안 됩니다. 만약 초과하는 순간이 발생하면, 그 이후에 어떤 문자가 오더라도 이미 짝이 맞지 않는 상태가 확정되므로 즉시 무효(NO) 처리가 가능합니다.

3. 상태 전이의 추적:
   - '('를 만나면: 처리해야 할 열린 괄호가 하나 늘어납니다. (Stack에 push 하거나 카운터 증가)
   - ')'를 만나면: 가장 최근에 열린 '(' 하나를 소모하여 짝을 맞춥니다. (Stack에서 pop 하거나 카운터 감소)

4. 보존 법칙 및 결론: 전체 문자열을 모두 확인했을 때, 소모되지 않고 남은 '('가 있다면 이는 짝을 찾지 못한 것이므로 무효입니다. 즉, 모든 처리가 끝난 시점에 잔여 '('가 0개여야만 최종적으로 "YES"가 성립합니다.

5. 효율적 대안: 괄호의 종류가 '('와 ')' 한 가지뿐이므로, 메모리를 더 사용하는 자료구조인 Stack 대신 정수형 변수(Counter) 하나만으로도 모든 상태를 완전하게 표현할 수 있습니다.

### 구현 세부 사항

- 테스트 케이스의 개수 $T$만큼 반복문을 실행합니다.
- 각 문자열을 순회하며 카운트를 조정하되, 카운트가 음수가 되는 즉시 반복을 탈출하고 "NO"를 출력합니다.
- 순회 종료 후 카운트가 0인지 확인하여 최종 결과를 도출합니다.

## 복잡도 분석

- 시간 복잡도: $O(T \times L)$
  - 테스트 케이스 수 $T$에 각 문자열의 길이 $L$을 곱한 만큼의 연산이 수행됩니다.
- 공간 복잡도: $O(1)$ (카운터 방식 사용 시) 또는 $O(L)$ (스택 사용 시)
  - 괄호의 개수를 저장하는 변수는 문자열 길이에 상관없이 일정한 공간을 차지합니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(T \times L)$)의 도출:
   각 테스트 케이스마다 주어진 문자열의 처음부터 끝까지 한 번의 선형 탐색을 수행합니다. 탐색 과정에서의 덧셈/뺄셈 및 조건문 판별은 상수 시간 $O(1)$에 해당하므로, 한 케이스당 $O(L)$의 시간이 소요됩니다. 이를 $T$번 반복하므로 전체 시간 복잡도는 $O(T \times L)$로 결정됩니다.

2. 공간 복잡도($O(1)$)의 도출:
   스택을 사용할 경우 최대 문자열 길이 $L$만큼의 공간이 필요하지만, 이 문제의 논리상 '현재 열려 있는 괄호의 개수'라는 누적 정보만 알면 되므로 단 하나의 정수형 변수만으로 상태 관리가 가능합니다. 입력 데이터를 제외한 알고리즘 자체의 추가적인 메모리 요구량은 입력 크기에 독립적이므로 $O(1)$이 됨을 알 수 있습니다.
