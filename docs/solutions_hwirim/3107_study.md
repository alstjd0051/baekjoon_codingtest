# BOJ 3107 - IPv6

## 문제 분석 및 풀이 전략

이 문제는 축약된 IPv6 주소를 원래의 128비트 형태(32자리 16진수, 8개 그룹)로 복원하는 문자열 처리 문제입니다. 두 가지 축약 규칙을 정확히 이해하고 역순으로 적용하는 논리적 과정이 필요합니다.

### 연역적 사고 과정

1. 문제의 구조 정의:
   - 표준 형태: `XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX` (8개 그룹, 각 그룹 4자리)
   - 축약 규칙 1: 각 그룹 앞자리의 0 생략 가능.
   - 축약 규칙 2: 0으로만 이루어진 연속된 그룹들을 `::`로 한 번 대체 가능.

2. 복원 전략 수립:
   - 단계 1: `::` 처리. 문자열 내에 `::`이 있다면, 이는 생략된 0 그룹들이 위치한 지점입니다. 전체 그룹이 8개가 되도록 부족한 만큼의 `0000` 그룹을 채워 넣어야 합니다.
   - 단계 2: 그룹별 복원. 콜론(`:`)을 기준으로 분리된 각 그룹이 4자리가 되도록 앞부분에 0을 채워 넣습니다. (예: `db8` -> `0db8`)

3. 세부 논리 유도:
   - 명령어 `split(':')`을 사용할 때, `::`이 양 끝에 있거나 중간에 있는 경우 빈 문자열(`''`)이 발생할 수 있음을 고려해야 합니다.
   - 특히 `::1` -> `['', '', '1']`, `1::` -> `['1', '', '']`, `1::1` -> `['1', '', '1']`과 같이 빈 문자열이 발생하는 상황을 분석하여 정확한 그룹 개수를 계산합니다.
   - 전체 그룹 개수가 8개가 되기 위해 필요한 빈 그룹의 개수는 `8 - (현재 유효한 그룹 개수)`입니다.

4. 최종 결합: 복원된 8개의 4자리 그룹을 다시 콜론으로 연결하여 출력합니다.

### 구현 세부 사항

- `ipv6.replace('::', ':zero_filler:')`와 같은 전처리 방식을 사용하거나, `split(':')` 결과를 순회하며 빈 문자열이 나타나는 지점에 필요한 만큼 0 그룹을 삽입하는 로직을 구현합니다.
- 각 그룹을 4자리로 만드는 작업은 `zfill(4)` 함수 혹은 문자열 포매팅을 활용합니다.

## 복잡도 분석

- 시간 복잡도: $O(L)$
  - 입력 문자열의 길이 $L$은 최대 39자이며, 이를 한두 번 순회하며 문자열을 분리하고 결합하는 작업은 상수 수준에 가까운 선형 시간 내에 완료됩니다.
- 공간 복잡도: $O(1)$ (혹은 결과 문자열의 크기에 비례)
  - 복원된 IPv6 주소의 길이는 항상 고정된 128비트(39자)이므로 입력을 제외한 추가 공간 사용량은 고정되어 있습니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(L)$)의 도출:
   입력 문자열을 읽고(1회), 콜론을 기준으로 분리하며(1회), 각 8개 그룹에 대해 0을 채우고 다시 합치는 과정은 모두 입력 길이 $L$ 또는 고정된 그룹 수 8에 비례합니다. $L$이 39로 매우 작으므로, 이는 사실상 즉시 처리 가능한 $O(L)$의 선형 시간 복잡도를 가짐을 알 수 있습니다.

2. 공간 복잡도($O(1)$)의 도출:
   우리가 생성하는 최종 문자열과 중간 과정에서 사용하는 리스트의 크기는 입력 문자열의 크기(최대 39자)와 결과의 크기(항상 39자)에 의해 제한됩니다. 데이터의 규모가 커져도 결과물은 항상 32자리 16진수와 7개의 콜론으로 고정되므로, 공간 복잡도는 상수 수준인 $O(1)$로 볼 수 있습니다.
