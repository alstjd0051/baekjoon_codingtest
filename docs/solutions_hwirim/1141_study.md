# BOJ 1141 - 접두사

## 문제 분석 및 풀이 전략

이 문제는 주어진 문자열 집합에서 어느 한 단어가 다른 단어의 접두사가 되지 않는 부분집합의 최대 크기를 구하는 문제입니다. 문자열 간의 포함 관계(접두사 관계)를 분석하여 최적의 선택을 하는 그리디(Greedy) 알고리즘 혹은 정렬을 기반으로 한 완전 탐색으로 해결할 수 있습니다.

### 연역적 사고 과정

1. 문제의 핵심 조건 분석:
   - 부분집합 내의 임의의 두 단어 $A, B$에 대해 $A$가 $B$의 접두사여서는 안 됩니다.
   - 만약 $A$가 $B$의 접두사라면, $A$와 $B$는 동시에 부분집합에 포함될 수 없습니다.

2. 관계의 방향성 파악:
   - 문자열 $A$가 $B$의 접두사가 되기 위해서는 반드시 $A$의 길이가 $B$의 길이보다 작거나 같아야 합니다.
   - 따라서 문자열들을 길이에 따라 오름차순으로 정렬하면, 뒤에 오는 긴 문자열이 앞에 오는 짧은 문자열의 접두사가 될 확률은 0입니다.

3. 그리디 선택의 정당성 유도:
   - $S_i$가 $S_j$ ($i < j$, 즉 $|S_i| \le |S_j|$)의 접두사라고 가정해 봅시다.
   - 이 경우 $S_i$와 $S_j$ 중 하나는 반드시 제외해야 합니다.
   - 이때 더 짧은 문자열인 $S_i$를 제외하는 것이 항상 유리하거나 최소한 손해는 아닙니다. 왜냐하면 $S_i$가 다른 문자열의 접두사가 될 가능성이 $S_j$보다 높기 때문입니다. (만약 $S_k$가 $S_j$를 접두사로 가진다면, 자동으로 $S_i$도 $S_k$의 접두사가 됩니다.)
   - 따라서 '나를 접두사로 가지는 더 긴 문자열이 존재한다면 나는 부분집합에서 빠진다'는 전략이 성립합니다.

4. 최종 알고리즘 수립:
   - 중복된 단어를 제거합니다 (동일한 단어는 상호 접두사 관계이므로 최대 1개만 포함 가능).
   - 단어들을 길이에 따라 정렬합니다.
   - 각 단어에 대해 자신보다 뒤에 있는(즉, 더 길거나 같은) 단어들 중 자신을 접두사로 포함하는 단어가 있는지 확인합니다.
   - 그런 단어가 하나도 없는 경우에만 카운트를 증가시킵니다.

### 구현 세부 사항

- `string.startswith(prefix)` 메서드를 활용하여 접두사 여부를 간단히 판별합니다.
- 정렬 후 이중 반복문을 통해 비교 리스트를 순회합니다.

## 복잡도 분석

- 시간 복잡도: $O(N^2 \times L)$
  - 중복 제거 및 정렬 후, 모든 단어 쌍($N^2$)에 대해 접두사 검사($L$, 문자열 길이)를 수행합니다. $N \le 50$으로 매우 작기 때문에 이는 충분히 효율적입니다.
- 공간 복잡도: $O(N \times L)$
  - 입력받은 모든 단어를 저장하기 위한 메모리가 필요합니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(N^2 \times L)$)의 도출:
   단어의 개수가 $N$일 때, 각 단어를 기준으로 삼아 나머지 모든 단어와 비교하는 이중 루프가 발생하므로 비교 횟수는 $N(N-1)/2$에 비례하는 $O(N^2)$입니다. 한 번의 비교 시 문자열의 접두사 여부를 확인하는 데 문자열의 최대 길이인 $L$만큼의 시간이 소요되므로, 총 시간 복잡도는 $O(N^2 \times L)$이 됩니다. $N=50, L=50$일 때 연산 횟수는 약 125,000번으로 극히 적은 수치입니다.

2. 공간 복잡도($O(N \times L)$)의 도출:
   단어 리스트를 보관하고 정렬된 리스트를 따로 관리하기 위해 단어 전체 총합 길이에 비례하는 공간이 필요합니다. 추가적인 대규모 공간을 요구하지 않으므로 전형적인 선형 공간 복잡도를 가집니다.
