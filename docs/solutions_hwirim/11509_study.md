# BOJ 11509 - 풍선 맞추기

## 풀이 전략

이 문제는 탐욕법(Greedy Algorithm)을 사용하여 해결할 수 있습니다. 최소한의 화살을 사용하기 위해 각 단계에서 최선의 선택을 하는 논리적 근거는 다음과 같습니다.

### 연역적 사고 과정

1. 모든 풍선은 반드시 터뜨려야 합니다. 풍선은 왼쪽에서 오른쪽으로 일렬로 배치되어 있으며, 화살 또한 왼쪽에서 오른쪽으로만 이동합니다. 따라서 왼쪽부터 순차적으로 풍선을 처리하는 것이 타당합니다.

2. 화살의 물리적 특성을 고려합니다. 높이 $H$에서 풍선을 터뜨린 화살은 소멸하지 않고 높이가 $H-1$이 되어 계속 오른쪽으로 진행합니다. 이는 기존에 쏜 화살이 미래의 풍선을 터뜨릴 수 있는 '자원'이 됨을 의미합니다.

3. 현재 위치에서 높이 $H$인 풍선을 만났을 때, 우리는 두 가지 선택을 할 수 있습니다.
   - 이미 높이 $H$로 날아오고 있던 화살이 있다면 해당 화살을 사용합니다.
   - 활용 가능한 화살이 없다면 새로운 화살을 높이 $H$에서 쏩니다.

4. 최적 부분 구조(Optimal Substructure) 관점에서, 이미 존재하는 화살을 사용하는 것은 새로운 화살을 추가로 소비하지 않으면서 풍선을 제거할 수 있으므로 항상 최선의 선택입니다. 새로운 화살을 쏘는 행위는 전체 화살 개수를 증가시키기 때문에, 기존 자원이 없을 때만 수행해야 합니다.

5. 결론적으로, 현재 날아오고 있는 모든 화살의 높이 상태를 관리하며, 왼쪽부터 풍선을 확인해 나갈 때 해당 높이의 화살 존재 여부에 따라 자원 재사용 또는 신규 투입을 결정하는 탐욕적 선택이 전체 문제의 최적해를 보장합니다.

### 구현 세부 사항

- 날아오고 있는 화살의 상태 관리: 높이의 최대 범위가 1,000,000이므로, 각 높이별 화살 개수를 저장할 수 있는 배열 자료구조를 사용합니다.
- 순회 및 갱신: 풍선 리스트를 순차적으로 확인하며 `arrows[H]`가 존재하면 `arrows[H]-1`, `arrows[H-1]+1` 처리를 수행합니다. 존재하지 않으면 전체 카운트를 증가시킨 후 `arrows[H-1]+1`을 수행합니다.

## 복잡도 분석

- 시간 복잡도: $O(N)$
  - 풍선의 개수($N$)만큼 한 번의 순회로 결과를 도출하며, 각 시점의 배열 접근은 $O(1)$입니다.
- 공간 복잡도: $O(K)$ ($K$는 풍선의 최대 높이)
  - 높이 정보의 범위만큼 배열 공간이 필요합니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(N)$)의 도출:
   우리는 입력받은 $N$개의 풍선 높이 정보를 왼쪽에서 오른쪽으로 한 번만 훑으면 됩니다. 각 풍선(원소)에 대해 수행하는 작업은 '현재 높이에 화살이 있는지 확인(배열 인덱스 접근)', '화살 개수 갱신(덧셈/뺄셈 연산)'뿐입니다. 이 모든 작업은 상수 시간($O(1)$) 내에 완료되므로, 전체 시간 복잡도는 입력 크기에 정비례하는 $O(N)$이 됨을 연역적으로 유도할 수 있습니다.

2. 공간 복잡도($O(K)$)의 도출:
   문제 조건에서 높이 $H$는 최대 1,000,000으로 주어집니다. 우리의 알고리즘은 각 높이 단계에서 날아오고 있는 화살의 개수를 '전역적으로' 추적해야 합니다. 이 상태를 유지하기 위해 가장 높은 풍선의 높이($K$)에 해당하는 인덱스를 가진 배열이 필요하므로, 메모리 사용량은 풍선의 개수 $N$이 아닌 높이의 범위 $K$에 종속됩니다. 따라서 공간 복잡도는 $O(K)$가 됩니다.
