# BOJ 1655 - 가운데를 말해요

## 문제 분석 및 풀이 전략

이 문제는 숫자가 실시간으로 추가될 때마다 현재까지 입력된 수들의 중간값(Median)을 빠르게 찾아야 하는 동적 중간값 탐색 문제입니다. 시간 제한이 0.1초로 매우 촉박하며 데이터 개수($N$)가 100,000개이므로, 매번 정렬하는 방식($O(N^2)$)으로는 해결이 불가능합니다.

### 연역적 사고 과정

1. 중간값의 정의와 성질: 데이터셋을 크기 순으로 나열했을 때 정중앙에 위치하는 값입니다. 이는 데이터를 '작은 수들의 집합'과 '큰 수들의 집합'으로 양분했을 때, 그 경계선에 있는 값들과 밀접한 관련이 있습니다.

2. 자료구조의 선택:
   - 배열/리스트: 삽입 시 정렬 상태를 유지하려면 $O(N)$이 소요되어 부적합합니다.
   - 힙(Heap): 삽입 시 $O(\log N)$이 소요되며, 최댓값이나 최솟값에 $O(1)$로 접근할 수 있습니다. 경계값을 관리하기에 최적입니다.

3. 두 개의 힙을 이용한 균형 관리:
   - 왼쪽 집합(Max-Heap): 현재까지의 수 중 작은 절반을 저장합니다. 루트는 이들 중 가장 큰 값이 됩니다.
   - 오른쪽 집합(Min-Heap): 현재까지의 수 중 큰 절반을 저장합니다. 루트는 이들 중 가장 작은 값이 됩니다.

4. 유지해야 할 핵심 규칙:
   - 크기 균형: Max-Heap의 크기는 Min-Heap의 크기와 같거나 1만큼만 커야 합니다. 이는 전체 개수가 홀수일 때 Max-Heap의 루트가 정확한 중간값이 되고, 짝수일 때 두 중간값 중 작은 값이 위치하게 하기 위함입니다.
   - 값의 역전 방지: 항상 Max-Heap의 루트는 Min-Heap의 루트보다 작거나 같아야 합니다. 만약 이 규칙이 깨진다면 두 루트값을 교환하여 경계를 재설정합니다.

5. 결과 도출: 위 규칙을 유지하며 숫자를 하나씩 넣을 때마다 Max-Heap의 루트를 출력하면, 그것이 항상 문제 조건에 부합하는 중간값이 됨을 논리적으로 보장할 수 있습니다.

### 구현 세부 사항

- `heapq` 모듈을 사용하되, 파이썬은 최소 힙만 지원하므로 Max-Heap은 값에 음수 기호를 붙여 관리합니다.
- `sys.stdin.readline`을 사용하여 대량의 입력을 0.1초 내에 처리할 수 있도록 최적화합니다.

## 복잡도 분석

- 시간 복잡도: $O(N \log N)$
  - 각 숫자($N$개)가 들어올 때마다 최대 2번의 힙 삽입($O(\log N)$)과 필요한 경우 힙 교체 연산이 수행됩니다.
- 공간 복잡도: $O(N)$
  - 입력받은 모든 숫자를 두 개의 힙에 나누어 저장해야 합니다.

### 복잡도 도출 사고 과정

1. 시간 복잡도($O(N \log N)$)의 도출:
   알고리즘은 $N$번의 삽입 및 조정 단계를 거칩니다. 각 단계에서는 힙에 요소를 추가하거나 최상단 요소를 제거하는 작업을 수행하는데, 힙의 높이가 데이터 개수에 로그 비례($\log N$)하므로 각 작업은 $O(\log N)$의 비용이 듭니다. $N$개의 데이터에 대해 로그 시간 작업을 반복하므로 전체 복잡도는 정렬과 동일한 수준인 $O(N \log N)$임을 연역적으로 알 수 있습니다. 이는 100,000개의 데이터에 대해 약 $10^5 \times 17 \approx 1.7 \times 10^6$번의 연산만 필요로 하므로 0.1초의 타이트한 제한 시간 내에 통과할 수 있는 유일한 대안입니다.

2. 공간 복잡도($O(N)$)의 도출:
   우리는 중간값을 지속적으로 계산하기 위해 지금까지 들어온 모든 숫자를 유실 없이 관리해야 합니다. 정렬된 상태를 유지하는 대신 두 개의 힙에 분산하여 저장하지만, 결과적으로 저장되는 총 원소의 개수는 $N$과 정확히 일치하므로 공간 복잡도는 입력 데이터의 크기에 비례하는 $O(N)$이 됩니다.
